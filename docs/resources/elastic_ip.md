# latitudesh_elastic_ip (Resource)

The elastic IP resource allows you to reserve and manage Elastic IPs â€” static public IPv4 addresses that can be assigned to a server and moved between servers within the same project â€” within your [Latitude.sh](https://latitude.sh/) account.

> **Note:** Elastic IP provisioning is asynchronous. On create the IP transitions `configuring â†’ active`; on move, `active â†’ moving â†’ active`; on destroy, `active â†’ releasing â†’ gone`. The provider waits for each transition with configurable timeouts (10 minutes each by default).

## Example usage

```hcl
data "latitudesh_region" "region" {
  slug = "SAO2"
}

resource "latitudesh_project" "project" {
  name              = "Production Environment"
  environment       = "Production"
  provisioning_type = "on_demand"
}

resource "latitudesh_server" "server" {
  hostname         = "example-host"
  operating_system = "ubuntu_24_04_x64_lts"
  plan             = "c2-small-x86"
  project          = latitudesh_project.project.id
  site             = data.latitudesh_region.region.slug
}

resource "latitudesh_elastic_ip" "elastic_ip" {
  project   = latitudesh_project.project.id
  server_id = latitudesh_server.server.id
}
```

To move the Elastic IP to a different server, change `server_id` and re-apply.

### Using the provider-level project default

Set `project` on the provider block and omit it on the resource:

```hcl
provider "latitudesh" {
  project = "proj_abc123"
}

resource "latitudesh_elastic_ip" "elastic_ip" {
  server_id = "sv_XYZ123"
}
```

### Customizing timeouts

Override the default 10-minute timeouts per operation. Note the `=` â€” `timeouts` is a nested attribute, not a block:

```hcl
resource "latitudesh_elastic_ip" "elastic_ip" {
  project   = latitudesh_project.project.id
  server_id = latitudesh_server.server.id

  timeouts = {
    create = "15m"
    update = "15m"
    delete = "5m"
  }
}
```
